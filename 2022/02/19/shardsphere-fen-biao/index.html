1 ShardSphere 简介

Sharding-Sphere是一套开源的分布式数据库中间件,是Apache的顶级项目,支持数据分片、读写分离，同时兼容多种数据库，通过可插拔架构，理想情况下，可以做到对业务代码无感知。
ShardingSphere下有两款成熟的产品：sharding jdbc和sharding proxy。

2 背景

系统上线后发现埋点表和接口日志表2天的数据量就已经300万的数据了,单表数据量太大且持续增长,查询操作非常耗时,决定使用分表中间件：ShardingSphere。

3 需求

  • 集成多数据源
  • 分表可配置分片周期
  • 动态建表
  • 支持分片表变更表结构

4 实现方式

  • 使用dynamic-datasource-spring-boot-starter组件集成多数据源,将不参与分表的数据源独立于Shardsphere之外,对分表的SQL使用sharding jdbc数据源，对不涉及到分表的SQL，使用普通数据源。
  • 使用标准分片策略对日期精准分片 ,对于范围日期条件,支持范围分片查询
  • 项目初始化缓存所有分片表名,对于分片表名,如果缓存匹配不上,需要新建分片表
  • 可在 td_activity_param 表配置分片表原始表结构,后台执行建表语句

集成多数据源

@Lazy  
@Resource(name = "shardingDataSource")  
AbstractDataSourceAdapter shardingDataSource;  

@Bean  
public DynamicDataSourceProvider dynamicDataSourceProvider() {  
    Map<String, DataSourceProperty> datasourceMap = dynamicDataSourceProperties.getDatasource();  
    return new AbstractDataSourceProvider() {  
        @Override  
        public Map<String, DataSource> loadDataSources() {  
            Map<String, DataSource> dataSourceMap = createDataSourceMap(datasourceMap);  
            dataSourceMap.put(SHARDING_DATA_SOURCE_NAME, shardingDataSource);  
            return dataSourceMap;  
        }  
    };  
}

@Primary  
@Bean  
public DataSource dataSource(DynamicDataSourceProvider dynamicDataSourceProvider) {  
    DynamicRoutingDataSource dataSource = new DynamicRoutingDataSource();  
    dataSource.setPrimary(dynamicDataSourceProperties.getPrimary());  
    dataSource.setStrict(dynamicDataSourceProperties.getStrict());  
    dataSource.setStrategy(dynamicDataSourceProperties.getStrategy());  
    dataSource.setProvider(dynamicDataSourceProvider);  
    dataSource.setP6spy(dynamicDataSourceProperties.getP6spy());  
    dataSource.setSeata(dynamicDataSourceProperties.getSeata());  
    return dataSource;  
}

public static void tableNameCacheReload(String schemaName) {  
    String tableExistKey = RedisKeyUtil.getTableExistList();  
    List<String> tableNameList = shardMapper.getAllTableNameBySchema(schemaName);  
    RedisUtil.KeyOps.delete(tableExistKey);  
    RedisUtil.ListOps.leftPushAll(tableExistKey, tableNameList);  
}

标准分片策略精准分片算法

@Override  
public String doSharding(Collection<String> availableTargetNames, PreciseShardingValue<Date> preciseShardingValue) {  
    Date date = preciseShardingValue.getValue();  
    logger.info("Sharding input:" + date);  
    String logicTableName = preciseShardingValue.getLogicTableName();  
    String suffix = getSuffixBySpecifyCycle(logicTableName, date);  
    return shardingTablesCheckAndCreatAndReturn(logicTableName, logicTableName.concat(suffix));  

}

获取分片表名

protected String getSuffixBySpecifyCycle(String logicTableName, Date date) {  
    Integer specifyInterval = ShardingAlgorithmTool.shardCycleMap.get(logicTableName);  
    if (Objects.isNull(specifyInterval)) {  
        specifyInterval = this.defaultCycle;  
    }  
    NumberFormat nf = NumberFormat.getInstance();  
    nf.setMinimumIntegerDigits(2);  
    Calendar calendar = Calendar.getInstance();  
    calendar.setTime(date);  
    int currentMonth = calendar.get(Calendar.MONTH) + 1;  
    int currentDate = calendar.get(Calendar.DAY_OF_MONTH);  
    int cycleDate = currentDate % specifyInterval == 0 ? (currentDate / specifyInterval - 1) * specifyInterval + 1 : (currentDate / specifyInterval) * specifyInterval + 1;  
    String concatTableName = "_".concat(nf.format(currentMonth)).concat("_").concat(nf.format(cycleDate));  
    return concatTableName;  
}

不存在自动建表

这里参考了双重检查的思想,存在则返回,不存在才加锁再次校验,避免并发重复建表的错误

public String shardingTablesCheckAndCreatAndReturn(String logicTableName, String resultTableName) {  
    logger.info("logicTableName:[{}],resultTableName:[{}]", logicTableName, resultTableName);  
    String tableExistKey = RedisKeyUtil.getTableExistList();  
    List tableExistList = RedisUtil.ListOps.range(tableExistKey, 0, -1);  
    if (tableExistList.contains(resultTableName)) {  
        return resultTableName;  
    }  

    String simpleUUID = IdUtil.fastSimpleUUID();  
    String lockKey = RedisKeyUtil.getTableExistKey();  
    try {  
        boolean lock = RedisUtil.LockOps.getLock(lockKey, simpleUUID);  
        if (lock) {  
            tableExistList = RedisUtil.ListOps.range(tableExistKey, 0, -1);  
            if (tableExistList.contains(resultTableName)) {  
                return resultTableName;  
            }  
            String createTableSql = shardMapper.selectTableCreateSql(logicTableName);  
            logger.info("createTableSql:{}", JSONObject.toJSONString(createTableSql));  
            createTableSql = createTableSql.replace("CREATE TABLE", "CREATE TABLE IF NOT EXISTS");  
            createTableSql = createTableSql.replace(logicTableName, resultTableName);  
            this.shardMapper.executeSql(createTableSql);  
            RedisUtil.ListOps.leftPush(tableExistKey, resultTableName);  
        }  
    } catch (Exception e) {  
        logger.error("建表异常:", e);  
    } finally {  
        RedisUtil.LockOps.releaseLock(lockKey, simpleUUID);  
    }  
    return resultTableName;  
}
